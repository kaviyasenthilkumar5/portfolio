Bug Report: BUG-001-LOGIN

Title: Login button remains disabled after entering valid credentials

Module: User Authentication

Severity: High

Priority: High

Status: Open

Environment

  • Browser: Chrome 127.0
  • OS: Windows 11
  • Application Version: 1.0.0-beta

Steps to Reproduce

  1. Navigate to the application login page (e.g., `https://example.com/login`).
  2. Enter valid username (e.g., `testuser@example.com`) in the 'Username' field.
  3. Enter valid password (e.g., `Password123!`) in the 'Password' field.
  4. Observe the 'Login' button.

Actual Result

The 'Login' button remains disabled even after entering both valid username and password, preventing the user from logging in.

Expected Result

The 'Login' button should become enabled once valid credentials are provided in both the 'Username' and 'Password' fields.
